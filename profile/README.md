# 蓝旭工作室 Bluemsun Office

这里是 **NENU（东北师范大学）蓝旭工作室** 的 GitHub 组织主页。

蓝旭工作室是一支面向校园真实场景进行产品设计、前端开发、后端开发、内容建设与工程实践的学生技术团队。我们关注校园服务、信息化建设、开源协作和技术传承，希望把在项目中积累的经验沉淀成可复用、可学习、可参与的开源资产。

在这里，你可以看到工作室正在维护或曾经沉淀的官网、业务系统、主题模板与组织文档。无论你是想了解蓝旭、参与项目、学习工程实践，还是寻找适合新手上手的贡献任务，都可以从这里开始。

## 如何参与

目前，蓝旭工作室各项目主要由组织成员维护，我们暂不接受非组织成员直接提交的 Pull Request。感谢大家对蓝旭开源项目的关注与支持，也欢迎通过这些方式了解和反馈：

- 阅读各仓库的 `README.md`，了解项目背景、启动方式和当前状态。
- 如果发现技术 Bug、文档错误、页面异常或其他可复现问题，可以在对应仓库提交 Issue。
- 关注本组织下对外开源的项目信息、更新动态和说明文档。
- 东北师范大学校内同学可以关注蓝旭工作室每年春季开放的纳新窗口，加入工作室后参与项目建设、技术实践与团队协作。

如果你提交 Issue，建议尽量说明问题所在仓库、复现步骤、期望表现、实际表现和相关截图或报错信息。清晰的问题反馈可以帮助维护者更快定位和处理。

## 建议补齐的开源规范

为了让新同学更容易参与，也让项目更便于长期维护，建议每个仓库逐步补齐：

- `README.md`：项目介绍、功能截图、技术栈、启动方式、目录结构、常见问题。
- `LICENSE`：明确项目授权方式。
- `CONTRIBUTING.md`：说明如何提交 Issue、组织成员如何开发、如何进行内部协作。
- Issue 模板：区分 Bug、功能建议、文档改进和新手任务。
- 项目状态：标注项目是否维护中、是否需要后端、是否有线上演示地址。
- 内部任务标签：为适合新成员熟悉项目的任务添加清晰标签，方便纳新后的同学逐步参与。

蓝旭工作室来自东北师范大学。我们相信，真实项目是最好的课堂，长期协作是最好的成长方式。
